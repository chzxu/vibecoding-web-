文件操作

8 条指令
ls
列出目录内容 —— 最常用的命令之一
📖 语法
ls [选项] [路径]
🔧 常用选项
-l 长格式显示（权限、大小、日期）
-a 显示所有文件（包括 . 开头的隐藏文件）
-h 人类可读大小（配合 -l 使用）
-t 按时间排序
-S 按大小排序
💡 示例
# 详细列出当前目录
$ ls -lah
drwxr-xr-x 5 user user 4.0K May 12 10:00 .
-rw-r--r-- 1 user user 256 May 12 09:30 notes.txt

# 按时间倒序
$ ls -lt
💡 ls -la 是你的"开盲盒"命令 —— 想看看目录里有什么，就用它！
cd
切换当前工作目录
📖 语法
cd [路径]
🔧 常用用法
cd ~ 回到 home 目录
cd / 回到根目录
cd .. 返回上一级目录
cd - 返回上一个目录
💡 示例
$ cd /var/log
$ cd ..
$ cd ~/Projects/my-app
💡 cd - 太实用了 — 在最后两个目录之间跳转，做运维的都知道！
pwd
显示当前工作目录的绝对路径
📖 语法
pwd
💡 示例
$ pwd
/home/zixu/projects/linux-course
💡 迷路时第一个想到的命令！pwd = print working directory。
mkdir
创建新目录
📖 语法
mkdir [选项] 目录名
🔧 常用选项
-p 递归创建父目录（不存在则自动创建）
-v 显示创建过程
💡 示例
# 创建单层目录
$ mkdir myproject

# 递归创建多层目录（不用一层层建了！）
$ mkdir -p projects/2026/05/笔记
💡 -p 参数是神技 — 建多层目录只用一条命令。
rm
删除文件或目录（⚠️ 危险！不可恢复）
📖 语法
rm [选项] 目标
🔧 常用选项
-r 递归删除（删除目录必加）
-f 强制删除，不提示
-rf ⚠️ 递归强制删除（慎用！）
💡 示例
# 删除文件
$ rm old-file.txt

# 删除目录（含所有内容）
$ rm -rf temp-folder
⚠️ rm -rf / 是最著名的"删库跑路"命令 —— 千万别乱试！（没有 -f 的话会一路问你确认）
cp
复制文件或目录
📖 语法
cp [选项] 来源 目标
🔧 常用选项
-r 递归复制（复制目录必加）
-i 覆盖前询问
-v 显示详细过程
-u 只复制更新的文件
💡 示例
# 复制文件
$ cp report.txt backup/

# 复制整个目录
$ cp -r my-project/ my-project-backup/
mv
移动文件 / 重命名文件
📖 语法
mv [选项] 来源 目标
🔧 常用选项
-i 覆盖前询问
-v 显示过程
💡 示例
# 重命名
$ mv old-name.txt new-name.txt

# 移动到其他目录
$ mv report.pdf ~/Documents/
💡 mv 即 rename — Linux 没有专门的重命名命令，mv 就是！
touch
创建空文件 / 更新文件时间戳
📖 语法
touch 文件名
💡 示例
# 创建空文件
$ touch notes.md index.html app.py

# 一次创建多个文件
$ touch {a,b,c}.txt → 创建 a.txt b.txt c.txt
💡 文件已存在时 touch 不会覆盖内容，只更新时间戳。超安全！

权限管理

5 条指令
chmod
修改文件或目录的权限
📖 语法
chmod [权限] 文件
🔧 权限表示法
数字法：chmod 755 → rwxr-xr-x
  r=4, w=2, x=1  所有者|组|其他人
符号法：chmod u+x → 给所有者加执行权限
💡 示例
# 数字法：所有者全部权限，组和其他只读+执行
$ chmod 755 script.sh

# 符号法：给所有人加执行权限
$ chmod +x run.sh
💡 755 = 脚本标准权限 | 644 = 普通文件标准权限 | 600 = 私密文件
chown
修改文件所有者（需 root）
📖 语法
chown [用户]:[组] 文件
💡 示例
# 修改所有者
$ sudo chown zixu config.yml

# 同时修改所有者和组
$ sudo chown zixu:developers app/
💡 用 -R 递归修改目录内所有文件：chown -R user:group dir/
chgrp
修改文件所属组
📖 语法
chgrp 组名 文件
💡 示例
$ sudo chgrp developers config.yml
$ chgrp -R www-data /var/www/
umask
设置新建文件的默认权限掩码
📖 语法
umask [掩码值]
💡 示例
# 查看当前 umask
$ umask
0022

# 设置新建文件默认为 755（目录）
$ umask 022
💡 umask = 777 - 目标权限。默认 022 意味着文件 644、目录 755。
sudo
以超级用户权限执行命令
📖 语法
sudo 命令
🔧 相关用法
sudo -i 切换到 root shell
sudo -u user 以指定用户身份执行
💡 示例
$ sudo apt update
$ sudo systemctl restart nginx
[sudo] password for zixu:
⚠️ 不是 root 用户！是"借权限执行一条命令"。sudo !! 用上一条命令加 sudo。

进程管理

6 条指令
ps
显示当前进程快照
📖 语法
ps [选项]
🔧 常用选项
ps aux 显示所有进程（BSD 风格）
ps -ef 显示所有进程（标准风格）
ps -u user 查看指定用户的进程
💡 示例
$ ps aux | grep nginx
root 12345 0.0 0.1 12345 6789 ? Ss 10:00 0:00 nginx: master
www-data 12346 0.0 0.1 12345 6789 ? S 10:00 0:00 nginx: worker
💡 ps aux | grep <进程名> — 查找进程的经典组合拳。
top
实时查看系统进程和资源占用
📖 语法
top
🔧 交互快捷键
q 退出
k 杀死进程（输入 PID）
M 按内存排序
P 按 CPU 排序
1 显示每个 CPU 核心
💡 top -u zixu 只查看你的进程。htop 是 top 的增强版（更漂亮），需要 apt install。
kill
发送信号终止进程
📖 语法
kill [信号] PID
🔧 常用信号
kill -15 SIGTERM — 礼貌请求退出（默认）
kill -9 SIGKILL — 强制杀死（⚠️ 无法被捕获）
kill -1 SIGHUP — 重新加载配置
💡 示例
# 先找到 PID
$ ps aux | grep firefox
zixu 23456 ... firefox
# 先试试礼貌退出
$ kill 23456
# 不行？上大招
$ kill -9 23456
💡 先 kill -15（给人家保存机会），不行再 kill -9。先礼后兵！
bg / fg
将进程放到后台/前台运行
📖 语法
command & 启动时直接放后台
Ctrl + Z 暂停当前进程
bg 将暂停的进程放到后台
fg 将后台进程调到前台
💡 示例
# 后台运行
$ python server.py &
[1] 12345

# 暂停后放后台
$ python long_task.py
(按 Ctrl+Z)
[2]+ 已停止 python long_task.py
$ bg
[2]+ python long_task.py &
💡 jobs 查看所有后台任务，fg %1 把任务 1 调到前台。
nohup
退出终端后继续运行命令
📖 语法
nohup 命令 &
💡 示例
# 关掉终端也不会停
$ nohup python train_model.py &
nohup: ignoring input and appending output to 'nohup.out'

# 指定输出日志
$ nohup ./backup.sh > backup.log 2>&1 &
💡 nohup + & = 在服务器上跑长时间任务的黄金搭档！
systemctl
管理系统服务（systemd）
📖 语法
systemctl [操作] [服务名]
🔧 常用操作
start 启动服务
stop 停止服务
restart 重启服务
enable 开机自启
status 查看状态
💡 示例
$ sudo systemctl start nginx
$ sudo systemctl enable docker
$ sudo systemctl status sshd
💡 装完软件第一步：systemctl enable --now service = 启动 + 开机自启一步到位！

网络相关

6 条指令
ping
测试网络连通性和延迟
📖 语法
ping [选项] 目标
🔧 常用选项
-c N 发送 N 次后停止
-i N 每 N 秒发一次
💡 示例
$ ping -c 4 baidu.com
PING baidu.com (39.156.66.10) 56(84) bytes of data.
64 bytes from 39.156.66.10: icmp_seq=1 ttl=50 time=4.23 ms
64 bytes from 39.156.66.10: icmp_seq=2 ttl=50 time=3.98 ms
💡 ping 通 = 网络没问题。Windows 下 ping 默认无限次，Linux 也是 — 加 -c 控制。
curl
传输数据工具（HTTP 请求瑞士军刀）
📖 语法
curl [选项] URL
🔧 常用选项
-o file 保存到文件
-L 跟随重定向
-H 自定义请求头
-d data POST 数据
-v 显示详细请求过程
💡 示例
# GET 请求看响应
$ curl https://api.github.com

# 下载文件
$ curl -o linux教程.pdf https://example.com/book.pdf

# POST JSON 数据
$ curl -X POST -H "Content-Type: application/json" -d '{"name":"zixu"}' https://api.example.com/users
💡 curl ifconfig.me 可以查看你的公网 IP！超实用。
wget
非交互式网络下载器
📖 语法
wget [选项] URL
🔧 常用选项
-O file 指定输出文件名
-c 断点续传
-r 递归下载
💡 示例
$ wget https://example.com/big-file.zip
$ wget -c https://example.com/large.iso # 续传
💡 curl vs wget：curl 更灵活（支持各种协议），wget 更擅长下载大文件（断点续传）。
netstat
查看网络连接、路由表、端口监听
📖 语法
netstat [选项]
🔧 常用选项
-tlnp 查看 TCP 监听端口（附进程名）
-ulnp 查看 UDP 监听端口
-an 显示所有连接
💡 示例
# 看谁在监听什么端口（排错必备）
$ sudo netstat -tlnp
Proto Recv-Q Send-Q Local Address Foreign Address State PID/Program
tcp 0 0 0.0.0.0:80 0.0.0.0:* LISTEN 1234/nginx
tcp 0 0 0.0.0.0:22 0.0.0.0:* LISTEN 567/sshd
💡 新系统用 ss 代替 netstat（更快），用法类似：ss -tlnp。
ssh
远程登录到另一台机器
📖 语法
ssh [选项] user@host
🔧 常用选项
-p port 指定端口（默认 22）
-i key 使用密钥文件登录
-v 调试模式
💡 示例
$ ssh admin@192.168.1.100
$ ssh -p 2222 -i ~/.ssh/id_rsa root@vps.example.com
💡 配置 ~/.ssh/config 可以免输地址和端口，用别名登录：
Host myserver
  HostName 1.2.3.4
  User root
  Port 2222
之后 ssh myserver 即可！
scp
基于 SSH 的安全文件传输
📖 语法
scp [选项] 来源 目标
💡 示例
# 本地文件传到服务器
$ scp myfile.zip user@server:/home/user/

# 从服务器下载文件
$ scp user@server:/var/log/nginx/access.log .

# 递归复制目录
$ scp -r project/ user@server:~
💡 -r = 目录传输。想更快？用 rsync 传输大项目（支持增量）。

文本处理

7 条指令
grep
在文件中搜索匹配的文本（文本搜索之王）
📖 语法
grep [选项] "模式" 文件
🔧 常用选项
-i 忽略大小写
-r 递归搜索目录
-n 显示行号
-v 反向匹配（不含模式的）
-c 只计数匹配行数
💡 示例
# 在日志中找 error
$ grep -i "error" /var/log/syslog

# 递归搜索所有 .py 文件中的函数定义
$ grep -rn "def " --include="*.py" .

# 排除注释行
$ grep -v "^#" config.conf
💡 grep -rn "关键词" . 是你最快找到代码位置的方法！grep 支持正则表达式。
sed
流编辑器 —— 文本替换、删除、插入
📖 语法
sed [选项] '命令' 文件
🔧 常用命令
s/旧/新/g 全局替换
/模式/d 删除匹配行
Nd 删除第 N 行
💡 示例
# 替换文本（输出到屏幕）
$ sed 's/old-text/new-text/g' file.txt

# 直接修改文件（用 -i）
$ sed -i 's/127.0.0.1/0.0.0.0/g' config.yml

# 删除空行
$ sed '/^$/d' messy.txt
💡 sed -i.bak 会在修改前备份原文件（生成 file.bak），强烈推荐养成习惯！
awk
强大的文本分析工具（按列处理）
📖 语法
awk '{print $N}' 文件
🔧 常用模式
$0 整行
$1 第 1 列
$NF 最后一列
NR 行号
-F, 指定分隔符（默认空格/制表符）
💡 示例
# 打印第 1 列和第 3 列
$ awk '{print $1, $3}' data.txt

# 以逗号分隔的 CSV：打印第 2 列
$ awk -F',' '{print $2}' users.csv

# 条件过滤：打印第 3 列大于 100 的行
$ awk '$3 > 100 {print $0}' scores.txt
💡 awk 其实就是 Linux 上的 Excel！按列处理数据，配合 grep/sed 三件套无敌。
cat
连接文件并输出到屏幕
📖 语法
cat [选项] 文件
🔧 常用选项
-n 显示行号
-b 只给非空行编号
💡 示例
# 查看文件
$ cat README.md

# 合并文件
$ cat part1.txt part2.txt > combined.txt
💡 文件太大？用 less 分页查看（less bigfile.log，按 q 退出）。
head / tail
查看文件的开头 / 结尾部分
📖 语法
head [-n 行数] 文件
tail [-n 行数] 文件
🔧 tail 特殊模式
tail -f 实时跟踪文件新增内容（看日志神器！）
💡 示例
# 看文件头 5 行
$ head -5 data.csv

# 看文件尾 10 行
$ tail -10 /var/log/syslog

# 实时追踪日志（运维必备）
$ tail -f /var/log/nginx/access.log
💡 tail -f 是运维人员的"直播"工具 — 实时看日志进来，按 Ctrl+C 退出。
sort
对文件内容排序
📖 语法
sort [选项] 文件
🔧 常用选项
-n 按数字排序
-r 反向排序
-u 去重（同 uniq）
💡 示例
# 按数字排序
$ sort -n scores.txt

# 先排序再去重再计数（经典组合）
$ sort tags.txt | uniq -c | sort -rn
wc
统计文件行数、单词数、字节数
📖 语法
wc [选项] 文件
🔧 常用选项
-l 只统计行数
-w 只统计单词数
-c 只统计字节数
💡 示例
$ wc -l app.py
231 app.py

# 代码行数统计
$ find . -name "*.py" | xargs wc -l | tail -1
1534 总行数
💡 wc -l 是统计文件行数最快的方式，没有之一！

包管理

4 条指令
apt
Debian/Ubuntu 软件包管理
📖 语法
sudo apt [操作] 包名
🔧 常用操作
update 更新软件源列表
upgrade 升级所有可升级的包
install pkg 安装软件包
remove pkg 移除软件包
search kw 搜索软件包
💡 示例
# 经典操作（先更新再安装）
$ sudo apt update && sudo apt install htop -y

# 搜索包
$ apt search web server
💡 习惯养成：sudo apt update 先于 sudo apt install — 源没更新可能装到旧版。
dpkg
Debian 包底层管理工具
📖 语法
dpkg [选项] .deb文件
🔧 常用操作
-i file.deb 安装本地 deb 包
-r pkg 移除包（保留配置）
-l 列出已安装包
-L pkg 查看包安装了哪些文件
💡 示例
$ sudo dpkg -i google-chrome.deb
$ dpkg -l | grep nginx
💡 dpkg 不处理依赖，装本地包后依赖用了 sudo apt install -f 修复。
yum / dnf
RedHat/CentOS/Fedora 包管理
📖 语法
sudo yum (或 dnf) [操作] 包名
🔧 常用操作
install nginx 安装
remove nginx 卸载
search kw 搜索
list installed 列出已安装
💡 示例
$ sudo yum install -y nginx
$ sudo dnf groupinstall "Development Tools"
💡 CentOS 8+ 用 dnf 替代 yum，命令用法几乎相同，但 dnf 更快。
pip
Python 包管理器（安装第三方库）
📖 语法
pip [操作] 包名
🔧 常用操作
install pkg 安装包
uninstall pkg 卸载
freeze 列出所有已安装包（生成 requirements.txt）
list 列出已安装包
💡 示例
# 安装库
$ pip install requests beautifulsoup4

# 导出依赖（项目必备）
$ pip freeze > requirements.txt
💡 强烈推荐用 venv 或 conda 创建虚拟环境，避免包冲突！
python -m venv myenv && source myenv/bin/activate

磁盘 / 存储

4 条指令
df
查看磁盘分区空间使用情况
📖 语法
df [选项]
🔧 常用选项
-h 人类可读格式（GB/MB）
-T 显示文件系统类型
💡 示例
$ df -h
Filesystem Size Used Avail Use% Mounted on
/dev/sda1 117G 45G 72G 39% /
/dev/sdb1 1.8T 1.2T 600G 67% /data
du
查看文件/目录占用的磁盘空间
📖 语法
du [选项] [路径]
🔧 常用选项
-h 人类可读
-s 只显示总计
--max-depth=N 只显示 N 层深度
💡 示例
# 当前目录总大小
$ du -sh
2.3G .

# 每个子目录占用（一级深度）
$ du -h --max-depth=1 | sort -rh
1.5G ./node_modules
456M ./dist
12M ./src
💡 du -sh * 看当前目录每个子项的大小，配合 sort -rh 按大小排序。
mount
挂载文件系统（U盘、ISO等）
📖 语法
mount [设备] [挂载点]
💡 示例
# 挂载 U 盘
$ sudo mount /dev/sdb1 /mnt/usb

# 挂载 ISO 镜像
$ sudo mount -o loop ubuntu.iso /mnt/iso
💡 用完记得 umount /mnt/usb（没拼错，是 umount 不是 unmount！）。
fdisk
磁盘分区管理工具
📖 语法
sudo fdisk [选项] /dev/sdX
💡 示例
# 查看分区表
$ sudo fdisk -l
Disk /dev/sda: 117.2 GiB
Device Boot Start End Sectors Size Id Type
/dev/sda1 * 2048 245596159 245594112 117G 83 Linux
💡 查看磁盘信息用 lsblk 更直观：树形结构显示所有磁盘和分区。

压缩 / 归档

4 条指令
tar
归档工具（打包 + 压缩，Linux 最常用）
📖 语法
tar [选项] 输出文件 输入文件
🔧 参数速记
c 创建   x 解压   z gzip   j bz2   v 显示过程   f 指定文件名
💡 示例
# 打包并压缩（.tar.gz）
$ tar -czvf project.tar.gz my-project/

# 解压 .tar.gz
$ tar -xzvf project.tar.gz

# 解压到指定目录
$ tar -xzvf project.tar.gz -C /tmp/
💡 记忆口诀：tar -czvf = 打包，tar -xzvf = 解包。
扩展名对应：.tar.gz → z，.tar.bz2 → j，.tar.xz → J
gzip
压缩单个文件（gunzip 解压）
📖 语法
gzip 文件   = 压缩（源文件替换为 .gz）
gunzip 文件.gz   = 解压
💡 示例
$ gzip biglog.txt
→ biglog.txt.gz 出现（原文件消失）

$ gunzip biglog.txt.gz
→ biglog.txt 恢复
💡 gzip -k 保留原文件（不删除）。zcat 可以直接查看 .gz 文件内容。
zip / unzip
跨平台压缩格式（Windows 通用）
📖 语法
zip [选项] 输出.zip 输入
unzip 文件.zip
🔧 常用选项
-r 递归压缩目录
-d dir 解压到指定目录
💡 示例
$ zip -r project.zip project/
$ unzip project.zip -d ./output
rsync
快速增量文件传输（本地/远程同步）
📖 语法
rsync [选项] 来源 目标
🔧 常用选项
-a 归档模式（保留权限、递归等）
-v 显示详情
-z 传输时压缩
--progress 显示进度
💡 示例
# 本地备份（增量同步）
$ rsync -avh /data /backup/ --progress

# 远程备份（通过 SSH）
$ rsync -avz site/ user@server:/var/www/
💡 rsync 比 scp 更聪明：只传输变化的部分，断点续传！备份神器。

用户管理

5 条指令
useradd
创建新用户
📖 语法
sudo useradd [选项] 用户名
🔧 常用选项
-m 创建 home 目录
-s /bin/bash 指定 shell
-G group 添加到附加组
💡 示例
# 创建用户并设置密码
$ sudo useradd -m -s /bin/bash alice
$ sudo passwd alice
New password:
💡 adduser（交互式）比 useradd 更友好，自动创建 home 目录等。Debian 系的特色。
passwd
修改用户密码
📖 语法
passwd [用户名]
💡 示例
$ passwd   # 改自己的密码
$ sudo passwd alice   # root 改别人密码
usermod
修改用户账户属性
📖 语法
sudo usermod [选项] 用户名
💡 示例
# 把用户加到 sudo 组（给管理员权限）
$ sudo usermod -aG sudo alice

# 锁定/解锁用户
$ sudo usermod -L bob   # 锁定
$ sudo usermod -U bob   # 解锁
userdel
删除用户
📖 语法
sudo userdel [-r] 用户名
💡 示例
# 删除用户
$ sudo userdel alice

# 删除用户同时删除 home 目录
$ sudo userdel -r alice
⚠️ -r 会删除用户的主目录和邮件池，确认后再操作。
groups / groupadd
管理用户组
📖 语法
groups [用户名]   # 查看用户属于哪些组
sudo groupadd 组名   # 创建新组
💡 示例
$ groups zixu
zixu : zixu sudo docker

$ sudo groupadd developers
$ sudo usermod -aG developers alice

系统信息

5 条指令
uname
查看系统内核信息
📖 语法
uname [选项]
🔧 常用选项
-a 显示所有信息
-r 内核版本
💡 示例
$ uname -a
Linux LAPTOP 6.6.87.2-microsoft-standard-WSL2 #1 SMP x86_64 GNU/Linux
free
查看内存使用情况
📖 语法
free [选项]
🔧 常用选项
-h 人类可读格式
-w 更细化的内存分类
💡 示例
$ free -h
total used free shared buff/cache available
Mem: 7.6G 2.1G 3.2G 0.1G 2.3G 5.0G
Swap: 2.0G 0.1G 1.9G
💡 看 available（可用内存）比 free 更有意义 — Linux 会用空闲内存做缓存。
lscpu
查看 CPU 详细信息
📖 语法
lscpu
💡 示例
$ lscpu
Architecture: x86_64
CPU op-mode(s): 32-bit, 64-bit
CPU(s): 16
Thread(s) per core: 2
Core(s) per socket: 8
Model name: AMD Ryzen 7 ...
lspci
列出所有 PCI 设备（显卡、网卡等）
📖 语法
lspci
💡 示例
$ lspci | grep -i vga
00:02.0 VGA compatible controller: Intel Corporation ...

# 详细显示
$ lspci -vv
dmesg
查看内核环形缓冲区的系统消息
📖 语法
dmesg [选项]
💡 示例
# 查硬件错误（U盘插入、驱动报错等）
$ dmesg | tail -20

# 实时跟踪内核消息
$ dmesg -w
💡 插了 U 盘没反应？dmesg | tail 看看内核有没有识别到！
