04 · 智能监督系统

让 AI 乖乖干活不跑偏

多层监督机制确保任务高效执行，防止死循环，智能恢复卡住状态。

👁️

执行监控（Beta）

自动检测重复工具调用（默认 5 次）、总调用次数（默认 10 次），评估智能体是否在朝任务目标前进。检测到异常时，顾问智能体自动介入给出替代策略。

📋

智能任务规划（Beta）

规划器在智能体开始工作前生成 3-7 个具体的可执行步骤。结合 Enricher 智能体分析完整上下文，制定 informed 的行动计划。

🚫

工具调用上限

硬性限制：普通智能体 100 次、受限智能体 20 次。接近上限时由 Reflector 引导智能体妥善结束任务。

🔄

Reflector 自动纠正

当 LLM 连续 3 次无法生成工具调用时自动介入。分析失败原因，引导智能体正确使用工具。

💡

开源模型优化建议

32B 以下模型建议同时开启执行监控 + 任务规划。实测 Qwen3.5-27B-FP8 搭配此功能后，结果质量提升 2 倍，但 token 消耗增加 2-3 倍。

⚡

链式摘要

智能管理 LLM 上下文：选择性总结旧消息，保留最新对话完整。防止 token 超限的同时保持对话连贯性。

# Linux amd64 快速安装 mkdir -p pentagi && cd pentagi wget -O installer.zip https://pentagi.com/downloads/linux/amd64/installer-latest.zip unzip installer.zip sudo ./installer # 安装器会自动完成： # 1. 检查 Docker、网络、系统要求 # 2. 生成 .env 配置文件 # 3. 配置 LLM 提供商（OpenAI / Anthropic / Ollama 等） # 4. 配置搜索引擎（DuckDuckGo / Tavily / Google 等） # 5. 生成安全凭证，启动 docker-compose # 也支持 Docker Compose 手动安装： curl -o .env https://raw.githubusercontent.com/vxcontrol/pentagi/master/.env.example # 编辑 .env 填写 API Key，然后： docker compose up -d
